# ZTF Practice — Zalo Tech Fresher Mock Exam Platform

[🇻🇳 Tiếng Việt](./README.md) · **🇬🇧 English**

A local web app for practising the **Zalo Tech Fresher** Entry Test — two mock exam types that mirror the real format:

- **Technical Practice** — 100 questions / 60 minutes / English — DSA, OOP, Networking, Database, OS, AI/ML
- **Zalo Aptitude (ZA)** — 30 questions / 70 minutes / Vietnamese — logic, number series, puzzles, word problems

Exams are generated by AI (Claude Code) using **style guides researched from real ZTF candidate writeups**, then **auto-graded** with a real-time countdown timer — same conditions as the actual proctored test.

## Quick demo

```bash
npm install
npm run dev   # → http://localhost:5173
```

The app boots with sample exams already in `src/exams/`. Pick one → click Start → answer questions → submit to see results plus history persisted in `localStorage`.

## Features

### Exam runner UI
- ⏱ **Countdown timer** matching the exam duration (60 or 70 minutes). When time runs out, the app auto-submits whatever answers are in.
- 🔖 **Flag questions** for later review before submitting.
- 🧭 **Sidebar navigation** — jump between questions, see which are answered / unanswered / flagged.
- 💾 **Auto-save while in progress** — reload the page mid-exam and you won't lose anything.
- ✅ **All-or-nothing scoring** for multi-correct questions (same as the real exam — no partial credit).
- 📊 **Results page**: total score, per-topic breakdown, list of incorrect answers with each question's `explanation`.
- 📚 **Attempt history** stored in `localStorage`, reviewable any time.

### AI-driven exam generation
- 🤖 **Slash commands `/gen-tech` and `/gen-za`** in Claude Code generate full exams (100 or 30 questions) in one shot.
- 🎨 **Style guides decoupled** (`prompts/ztf-*-style.md`) — phrasing patterns, distractor styles, and paraphrased samples to make generated questions feel closer to the real ZTF exam.
- 🔍 **Runtime JSON schema validation** — malformed exam files are skipped with a console warning, never crash the app.
- ♻️ **Vite HMR** — dropping a new JSON file into `src/exams/` makes it appear immediately.

## Generating new exams with Claude Code

In a terminal at the project root:

```bash
claude

# Generate a new tech exam (100 q, EN)
> /gen-tech

# Generate with a hint
> /gen-tech focus more on AI and database

# Generate a new ZA exam (30 q, VI)
> /gen-za

# Generate with a hint (in Vietnamese, matching the exam language)
> /gen-za tập trung puzzle ràng buộc
```

Each command will:
1. Read `prompts/schema.md` for the JSON shape
2. Read `prompts/generate-*.md` for topic distribution and quality rules
3. Read `prompts/ztf-*-style.md` for phrasing/distractor patterns and samples
4. Generate all 100/30 questions in one pass
5. Write to `src/exams/{type}/exam-{NNN}.json` with the next available NNN
6. Report topic distribution and single/multi counts

Vite HMR picks up the new file automatically — refresh the exam picker and it's there.

### Generating from the UI

The project ships with an `/api/generate` middleware in `vite.config.ts`. Clicking the **Generate** button in the UI triggers generation via the local Claude Code CLI (requires Claude Code installed locally).

## The style guide system

To make generated questions feel close to the real ZTF, this project uses **research-driven generation**:

1. **One-shot research pass** — collected format, phrasing, and sample patterns from public ZTF candidate writeups (VOZ, daynhauhoc, Zalo Careers blog).
2. **Style guides as markdown** (`prompts/ztf-tech-style.md`, `prompts/ztf-za-style.md`) — each file contains:
   - Phrasing and distractor patterns per topic
   - 8 paraphrased sample questions (schema-compatible)
   - Source citations
3. **Gen prompts reference the style guides** — `generate-technical.md` and `generate-za.md` open with a paragraph instructing Claude to read the style guide first.
4. **Format / topic / timing is authoritative** — sourced directly from Zalo's communication to candidates. Style guides shape *how* questions read, never *how many* or *which topics*.

The full spec and implementation plan are at `docs/superpowers/specs/` and `docs/superpowers/plans/`.

> **Note**: Sample questions in the style guides are **paraphrased reconstructions**, NOT verbatim ZTF questions. The actual ZTF exam is proctored and never published. The samples are calibration anchors only — they teach the gen LLM what real questions *look* and *feel* like.

## Exam JSON schema

Single source of truth: `prompts/schema.md`. Quick reference:

```jsonc
{
  "id": "tech-001",
  "type": "technical",            // "technical" | "za"
  "title": "Technical Practice #1",
  "language": "en",               // "en" | "vi"
  "durationMinutes": 60,
  "totalQuestions": 100,
  "passingScore": 70,             // optional, %
  "createdAt": "2026-05-26",
  "topics": ["DSA", "OOP", "..."],
  "questions": [
    {
      "id": "q1",
      "topic": "DSA",
      "type": "single",            // "single" | "multi"
      "prompt": "What is...",
      "code": "...optional code snippet...",
      "options": [
        { "id": "a", "text": "..." },
        { "id": "b", "text": "..." }
      ],
      "correct": ["b"],            // always an array
      "explanation": "..."         // 2–5 sentences, required
    }
  ]
}
```

The runtime validator in `src/lib/examLoader.ts` skips malformed files with a console warning instead of crashing.

## Scoring rules

| Situation | Score |
|---|---|
| Single-correct, right answer | 1 |
| Single-correct, wrong answer | 0 |
| Multi-correct, every right id selected, no extras | 1 |
| Multi-correct, missing or extra ids | 0 (all-or-nothing) |
| Unanswered | 0 (counts as wrong) |
| Time runs out | Auto-submit with current selections |

No partial credit. The real ZTF exam scores the same way.

## localStorage namespace

Every key is prefixed with `ztp:` (Zalo Tech Practice):

| Key | Contents |
|---|---|
| `ztp:attempts` | `AttemptResult[]` — full history across all exams |
| `ztp:inProgress:<examId>` | Snapshot of the current in-progress attempt (auto-saved on every answer change). Deleted on submit. |

Reload mid-exam → the app prompts "Resume in-progress attempt?".

To reset state → DevTools → Application → Local Storage → delete keys starting with `ztp:`.

## Contributing / extending

- **Add an exam by hand**: drop a schema-compliant JSON into `src/exams/{technical|za}/`. Vite picks it up.
- **Change scoring rules**: edit `src/lib/scoring.ts` (has unit tests).
- **Change UI**: components are plain React + inline-ish CSS.
- **Change topic distribution**: edit the table in `prompts/generate-*.md`.
- **Update style guides**: edit `prompts/ztf-*-style.md`. You can re-run research if Zalo's format changes.
